# Piecewise — a mobile-first jigsaw puzzle PWA

A jigsaw puzzle game built with React + TypeScript + Vite, rendered on Canvas for performance, with realistic interlocking pieces, touch/pinch/pan gestures, autosave, and offline PWA support.

## Getting started

```bash
npm install
npm run dev       # local dev server
npm run build     # production build -> dist/
npm run preview   # preview the production build
```

Requires Node 18+.

## What's implemented

- **Home gallery** with category filters (Nature, Animals, Love, Cities, Abstract) and 8 built-in original vector illustrations (no external image downloads, so they work fully offline).
- **Upload your own photo** (JPG/PNG/WebP), with automatic downsizing in a Web Worker (`utils/imageResize.worker.ts`) and a touch-friendly crop tool (drag to pan, pinch/buttons to zoom, rotate, 4 aspect ratios) in `components/ImageCropper.tsx`.
- **Difficulty picker**: 3x3 up to 12x12 (9-144 pieces) with Easy/Medium/Hard/Expert labels.
- **Canvas puzzle engine** (`engine/`): deterministic seeded piece-shape generation so tabs/blanks always interlock correctly (`shapeGenerator.ts`, `piecePath.ts`), drag/snap/rotate logic decoupled from rendering (`PuzzleEngine.ts`).
- **Gesture system** (`canvas/usePuzzleCanvas.ts`): Pointer Events (not separate mouse/touch handlers), one-finger drag-to-place, one-finger pan, two-finger pinch-zoom + pan, `touch-action: none`, a continuous `requestAnimationFrame` render loop, DPR-aware sizing via `ResizeObserver`, haptic feedback on snap.
- **Workspace tools**: shuffle, auto-arrange (tidies loose pieces into a grid), zoom in/out/reset, ghost-image guide toggle, full preview modal, fullscreen, pause/resume, restart, live timer/move-counter/progress bar.
- **Autosave & resume**: progress is written to `localStorage` every few seconds and on visibility/pagehide, and is restored automatically if you come back to the same image + difficulty.
- **Completion celebration**: canvas confetti, final time/moves, "Play again", and a Web Share / clipboard-copy "Share result".
- **Settings**: dark mode, high-contrast mode, optional piece rotation, classic vs. square piece edges, sound-effect toggle (toggle is wired up for a future audio pass - see Known simplifications).
- **Accessibility**: 44px+ touch targets, `aria-label`s on the canvas and controls, keyboard-reachable buttons with visible focus rings, a high-contrast mode, `prefers-reduced-motion` handling.
- **PWA**: installable, offline-capable app shell + built-in puzzle assets via `vite-plugin-pwa` (Workbox), manifest + maskable/standard icons in `public/icons` (generated by `scripts/make_icons.py`).

## Key scoping decisions

A few deliberate simplifications were made to keep the implementation robust and shippable rather than half-finished in more areas:

- **Piece tray = the scattered canvas area**, not a separate DOM list. All loose pieces live in the same pannable/zoomable world as the board (this is how most performant canvas jigsaw implementations work). "Auto-arrange" tidies them into a literal grid below the board; "Shuffle" scatters them again.
- **Snap-to-home-slot rather than neighbor-to-neighbor grouping.** Pieces snap into their correct final position when dropped close enough; they don't bind to each other if you connect two loose pieces away from the board. This keeps the drag/hit-testing model simple while still requiring correctly-interlocking shapes (which are real - see `piecePath.ts`).
- **Rotation is off by default** (toggle in Settings) and is applied via a "select a piece, then tap rotate" interaction rather than a one-finger twist gesture, since a reliable single-finger rotate gesture would conflict with the drag gesture.
- **No background music / multiplayer / leaderboards / daily challenge** - these were listed as "bonus"/"future" in the brief and are left out so the core mobile puzzle experience could be fully built and polished. The sound-effect toggle exists in Settings but only haptics are wired up today; dropping in `<audio>` cues for snap/complete would be a small follow-up.
- **Built-in images are original flat-vector illustrations**, not photos - this means the gallery ships fully offline with zero licensing concerns and a small bundle size.

## Project structure

```
src/
  engine/      pure puzzle logic: shapes, paths, drag/snap, persistence (no React, no Canvas API calls beyond hit-testing)
  canvas/      rendering + gesture hook that drives the <canvas>
  components/  reusable UI (cards, toolbar, modals, cropper, settings sheet)
  pages/       the 4 screens: Home, Upload, Difficulty, Workspace
  hooks/       timer, image loading, the top-level game/session hook
  store/       small context for navigation + persisted settings
  utils/       storage, formatting, difficulty table, image resize worker
  assets/puzzles/  built-in SVG illustrations + manifest
```
